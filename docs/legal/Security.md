# 甘农生活安全说明

- 文档版本：1.0
- 生效日期：2026 年 7 月 13 日
- 最后更新：2026 年 7 月 13 日
- 适用应用：甘农生活（`com.tika.gsaulife`）

## 1. 安全范围

本说明介绍官方源代码和官方 Release 的安全设计、已知边界与漏洞报告方式。它不构成绝对安全保证，也不覆盖未经审核的第三方修改版、Root 模块、重签名安装包或学校系统自身的安全性。

应用需要处理校园卡请求字段、付款字符串、学校 Cookie 和教务缓存。此类数据具有账户和个人信息属性，应避免截图公开、日志分享或交给不可信应用。

## 2. 本地存储

### 2.1 Android 应用沙箱

校园卡、学校会话、教务缓存和设置保存在 Android 应用私有目录。正常情况下，其他普通应用无法读取该目录。

应用设置：

- `android:allowBackup="false"`
- `android:fullBackupContent="false"`
- 云备份全部数据域排除
- 设备迁移全部数据域排除

应用没有实现开发者云同步，也不会把本地数据提交到 GitHub 仓库或 Cloudflare 存储。

### 2.2 应用层加密状态

当前版本使用普通 SharedPreferences 和 Android WebView Cookie 存储，没有使用独立密钥对校园卡字段、Cookie 和教务缓存进行应用层加密。

这意味着安全性依赖 Android 沙箱、锁屏、文件权限和设备完整性。以下环境可能绕过隔离：

- 已 Root 或解锁并授予高权限的设备。
- 存在系统漏洞、恶意调试工具或高权限恶意软件的设备。
- 被他人取得且未设置可靠锁屏的设备。
- 厂商或第三方备份工具绕过标准 Android 备份规则的环境。

不建议在无法信任的设备、公共设备或长期借用设备上保存学校会话和校园卡。

## 3. 网络通信

### 3.1 学校数据请求

原生数据请求使用以下 HTTPS 地址：

- `https://yktapp.gsau.edu.cn`
- `https://jwgl.gsau.edu.cn`
- `https://xgfw.gsau.edu.cn`

教务和学工数据客户端关闭自动重定向。接口返回 `3xx` 时，应用按会话失效处理，避免手工设置的学校 Cookie 被自动带到其他主机。

应用依赖 Android 系统信任库验证 TLS 证书，当前没有实现证书固定。系统信任库、用户安装证书、代理软件和设备系统安全状态会影响连接可信度。

### 3.2 WebView 登录

学校登录在 Android WebView 中完成。登录成功后，应用读取目标学校域名的 Cookie，并保存用于数据查询的会话字段。

登录完成地址会严格校验 HTTPS 协议、目标主机和路径。设置页重新登录会清除 WebView Cookie，避免沿用旧身份。应用不保存用户在学校页面输入的密码。

学校页面及其加载的资源由学校控制。学校统一认证、WebView 或页面资源出现安全问题时，应用无法独立修复服务端行为。

### 3.3 更新检查

更新功能的网络路径为：

- 应用向 `https://gh.re-tikara.fun` 请求最新 Release 信息。
- Cloudflare Worker 向 `https://api.github.com` 请求公开 Release API。
- 用户打开发布页时，Worker 向 `https://github.com/RE-TikaRa/GSAULife/releases` 转发页面请求。

更新检查只使用 Worker 预先列出的 Release API 路径。Worker 使用的 GitHub Token 不发送给客户端。

更新查询不附带校园卡、学校 Cookie、教务缓存或本地设置。Cloudflare 和 GitHub 仍会处理网络连接产生的 IP 地址与请求元数据。

## 4. 付款码安全

- 付款码图像由 ZXing 在本地内存中生成。
- 应用不把二维码图片保存到文件或相册。
- 付款字符串会短时缓存在校园卡本地记录中。
- 应用和桌面组件不会继续展示超过 60 秒的旧码。
- 全屏付款页保持屏幕常亮并提升当前窗口亮度。
- 设备截图、录屏、投屏和旁观者仍可能取得二维码，应用无法阻止所有系统级捕获方式。

付款码应视为敏感支付凭据。不要把二维码截图、校园卡链接或 `openid` 公开到 GitHub Issues、群聊和论坛。

## 5. 会话与缓存安全

- 教务和学工 Cookie 分开保存和更新。
- 会话带有本地版本号，登录状态改变后，旧请求不会覆盖新会话数据。
- 已取消或已经过时的刷新任务不会继续写入教务缓存。
- 教务 HTML 缺少数据表或当前学期标记时，不会覆盖有效缓存。
- 学工排名会校验成功码、关键字段和排名范围。
- 主动退出教务身份会清除两套会话、WebView Cookie、教学周设置和教务缓存。

离线缓存用于可用性，不代表学校当前最终数据。涉及考试、成绩和学业状态的重要事项仍需在官方系统确认。

## 6. Android 权限

| 权限 | 安全边界 |
|---|---|
| `INTERNET` | 访问学校 HTTPS 服务和更新服务 |
| `ACCESS_NETWORK_STATE` | 获取网络连接状态，不读取网络内容 |
| `POST_NOTIFICATIONS` | 显示持续刷新通知，可由用户撤销 |
| `FOREGROUND_SERVICE` | 在用户选择持续刷新时运行可见服务 |
| `FOREGROUND_SERVICE_DATA_SYNC` | 标识前台服务的数据同步用途 |
| `RECEIVE_BOOT_COMPLETED` | 设备启动后更新组件状态并清理旧码 |
| `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` | 到期后更新组件付款码显示状态 |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 用户主动前往系统页面授予电池优化例外 |

应用没有申请联系人、短信、通话、位置、相机、麦克风、存储或无障碍权限。

## 7. 发布与签名

官方 Release 通过 GitHub Actions 构建。发布流程执行：

1. 单元测试与 Android Lint。
2. Debug 与签名 Release 构建。
3. `apksigner` 签名验证。
4. 签名证书 SHA-256 摘要核对。
5. 创建 GitHub Release 并上传 APK。

当前工作流期望的签名证书 SHA-256 摘要为：

```text
3152994da0d1c7fb1e6129f23446cdc7baaf03fdea9160899a5dcc59f33446d9
```

安装包来源应为项目 GitHub Releases。第三方镜像、群文件、重签名 APK 和修改版可能使用不同证书并包含额外代码。

## 8. 日志与调试

应用没有接入远程崩溃上报或遥测平台。Android 系统日志、WebView 调试环境、代理抓包工具和开发者自行添加的日志仍可能显示请求错误或运行信息。

提交问题前应检查并删除：

- 校园卡链接与 `openid`。
- 付款字符串和二维码。
- 学校 Cookie 与请求头。
- 学号、姓名、卡号、成绩和排名。
- 包含账号信息的截图或录屏。

## 9. 已知安全边界

项目无法保证：

- Root、恶意系统镜像或高权限恶意软件中的数据保密性。
- 学校系统、统一认证、校园卡服务的安全性和持续可用性。
- 用户主动安装的 CA 证书、VPN、代理或抓包软件不会读取流量。
- 第三方修改版与官方源代码保持一致。
- Android 厂商后台机制严格遵守标准组件和备份行为。
- 屏幕内容不会被系统截图、录屏、投屏或物理观察取得。

## 10. 漏洞报告

发现安全问题时，优先通过以下方式报告：

- GitHub Security Advisory：在仓库的 Security 页面创建私密报告。
- 邮箱：163mail@re-tikara.fun

报告建议包含受影响版本、设备与 Android 版本、复现步骤、影响范围和必要日志。请勿发送真实校园卡链接、有效 Cookie、付款码、学校密码或无关个人信息；可使用已经失效或经过替换的示例数据。

项目按个人开源项目的能力处理报告，不承诺漏洞赏金、固定响应时间或商业级支持。
