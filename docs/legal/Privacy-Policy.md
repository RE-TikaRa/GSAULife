# 甘农生活隐私政策

- 文档版本：1.0
- 生效日期：2026 年 7 月 13 日
- 最后更新：2026 年 7 月 13 日
- 适用应用：甘农生活（`com.tika.gsaulife`）

## 1. 说明

甘农生活是一款面向甘肃农业大学学生的个人开源 Android 工具，用于展示校园卡付款码、查询教务信息、保存离线缓存和管理应用设置。

本应用没有用户注册系统、广告系统、统计分析系统或开发者业务后台。应用仅在用户主动使用相关功能时，向学校官方系统发送完成请求所需的数据。校园卡信息、学校账户、Cookie、成绩、课表、考试、排名和本地设置不会发送给开发者。手动检查更新会访问 Cloudflare Worker 与 GitHub，但不附带这些校园数据。

本政策中的“处理”包括应用在设备本地读取、解析、展示、保存和删除数据。应用在本地处理数据，不等于开发者取得或收集这些数据。

## 3. 应用处理的数据

| 数据类别 | 具体内容 | 用途 | 存储位置 |
|---|---|---|---|
| 校园卡链接字段 | `openid`、卡片编号 `id` | 向学校校园卡页面请求对应付款码 | Android 应用私有存储 |
| 校园卡资料 | 姓名、卡号、余额、本机备注 | 展示和区分多张校园卡 | Android 应用私有存储 |
| 付款码缓存 | 付款字符串、获取时间 | 在应用和桌面组件中短时展示二维码 | Android 应用私有存储；二维码图像仅在内存中生成 |
| 教务会话 | 教务系统 Cookie | 保持教务系统登录并查询成绩、课表和考试 | Android 应用私有存储与 Android WebView Cookie 存储 |
| 学工会话 | 学工系统 Cookie | 保持学工系统登录并查询学业排名 | Android 应用私有存储与 Android WebView Cookie 存储 |
| 成绩数据 | 学期、课程、成绩、学分、绩点、课程类型、考试类型、成绩明细 | 展示成绩并支持离线查看 | Android 应用私有存储 |
| 课表数据 | 学期、课程、教师、教室、星期、节次、周次 | 展示课表并支持离线查看 | Android 应用私有存储 |
| 考试数据 | 学期、课程、时间、地点、座位 | 展示考试安排并支持离线查看 | Android 应用私有存储 |
| 排名数据 | 学年、学期、专业排名、专业人数、班级排名、班级人数 | 展示学业排名并支持离线查看 | Android 应用私有存储 |
| 教学周设置 | 学期、开学日期序号、校准周次 | 计算当前教学周 | Android 应用私有存储 |
| 应用设置 | 外观模式、校园卡刷新方式、已同意法律文档版本 | 恢复用户选择并记录协议确认状态 | Android 应用私有存储 |
| 更新信息 | GitHub Release 版本号、更新说明、发布页地址 | 手动检查新版本 | 请求期间在内存中处理，不写入个人档案 |

应用不会主动读取通讯录、短信、通话记录、照片、文件、精确位置、麦克风、摄像头、设备序列号、广告标识符或其他应用的数据。

## 4. 数据来源

应用处理的数据来自以下来源：

1. 用户粘贴的甘肃农业大学校园卡付款码页面链接。
2. 用户在 Android WebView 中完成学校登录后，由学校系统设置的 Cookie。
3. 用户主动查询时，甘肃农业大学校园卡、教务和学工系统返回的数据。
4. 用户在应用内设置的校园卡备注、主题、刷新方式、开学日期和教学周。
5. 用户手动点击“检查更新”后，GitHub Releases 返回的公开发布信息。

## 5. 数据用途

应用仅在实现下列功能所需的范围内处理数据：

- 获取并展示校园卡付款码、姓名、卡号和余额。
- 在多张校园卡之间切换并保存本机备注。
- 登录学校系统并查询成绩、课表、考试安排和学业排名。
- 保存查询结果，供网络不可用或会话过期时离线查看。
- 根据开学日期或用户输入的当前周计算教学周。
- 根据用户选择显示浅色、深色或跟随系统的界面。
- 根据用户选择按需刷新或持续刷新校园卡付款码。
- 在用户主动操作后查询应用新版本。

应用不会使用上述数据进行广告投放、用户画像、商业营销、数据交易、自动决策或与本应用功能无关的分析。

## 6. 网络通信与接收方

### 6.1 甘肃农业大学官方系统

当用户使用对应功能时，应用会与以下学校服务通信：

| 服务 | 发送内容 | 返回内容 |
|---|---|---|
| 校园卡系统 | `openid`、卡片编号及常规 HTTPS 请求信息 | 付款字符串、姓名、卡号、余额 |
| 教务系统 | 教务 Cookie、查询参数及常规 HTTPS 请求信息 | 成绩、课表、考试安排页面 |
| 学工系统 | 学工 Cookie、查询参数及常规 HTTPS 请求信息 | 学业排名数据 |
| 学校统一认证 | 用户在 WebView 中填写的登录信息 | 登录结果与学校 Cookie |

登录账号和密码由学校登录页面接收。应用不会把用户填写的学校密码保存到自己的 SharedPreferences，也不会把密码发送给开发者。

### 6.2 GitHub 与 Cloudflare

用户点击“检查更新”时，应用会访问 `gh.re-tikara.fun` 的 Cloudflare Worker，再读取 GitHub Releases 的公开信息。应用不会在该请求中附带校园卡信息、学校 Cookie、教务数据或本地设置。

网络连接会自然包含 IP 地址、请求时间、TLS 和 HTTP 元数据。Cloudflare、GitHub、网络运营商及其基础设施可能按照各自政策处理这些信息。项目 Worker 代码不建立用户数据库，也不把更新请求写入应用业务存储。

## 7. 本地存储与保留期限

应用使用 Android SharedPreferences 和 WebView Cookie 存储保存数据，均位于当前应用的私有目录。当前版本没有把上述数据写入开发者服务器、GitHub 仓库、Cloudflare KV、对象存储或其他云端数据库。

数据通常保留至以下任一事件发生：

- 用户删除对应校园卡。
- 用户在设置页退出教务身份。
- 应用刷新或清理已经失效的付款码缓存。
- 用户在 Android 系统设置中清除应用数据。
- 用户卸载应用。

付款码图像由 ZXing 在内存中生成，不保存为图片文件。付款字符串可能作为校园卡本地缓存的一部分保存；超过有效展示时间后，应用和组件不会继续显示该旧码。

## 8. 系统备份与设备迁移

应用清单设置了 `android:allowBackup="false"`，并在 Android 数据提取规则中排除了云备份和设备迁移支持的全部数据域，包括文件、数据库、SharedPreferences、外部文件和设备保护存储。

这些配置用于阻止 Android 备份与迁移机制复制应用数据。设备厂商、系统修改、Root 环境或外部备份工具可能绕过标准 Android 行为，开发者无法控制此类环境。

## 10. 用户的控制与删除方式

用户可以通过以下方式管理数据：

- 长按校园卡，删除单张校园卡及其本地付款码缓存。
- 在设置页点击“退出教务身份”，清除教务、学工会话、教学周设置和全部教务缓存。
- 在 Android 系统设置中清除“甘农生活”的存储数据，删除应用保存的全部本地数据。
- 卸载应用，删除应用私有目录中的全部数据。
- 撤销通知权限、取消电池优化例外或关闭自启动权限。

由于开发者不持有应用内的校园卡、学校会话和教务缓存，开发者无法远程查看、导出或代替用户删除这些本地数据。

## 11. 数据安全

应用依赖 Android 应用沙箱保护本地数据，学校数据接口使用 HTTPS，原生学校请求不会携带 Cookie 自动跟随重定向。Release 工作流会验证发布 APK 的签名证书。

当前本地 SharedPreferences 没有增加独立的应用层加密。Root、恶意调试、系统漏洞、已解锁设备或能够读取应用私有目录的程序可能取得本地数据。详细说明见[安全说明](Security.md)。

## 12. 未成年人

本应用面向具备甘肃农业大学校园卡或学校账号的用户，不以未满十四周岁的儿童为目标，也不会建立儿童用户档案。未成年人应在监护人指导下使用本应用和学校账号。

## 13. 政策更新

当应用的数据处理方式、网络服务或功能发生实质变化时，本政策会随代码更新。文档顶部的“最后更新”日期用于标识当前版本。历史版本可以通过 Git 提交记录查看。
