# 第一版验收标准

甘农生活第一版以能够发布和日常使用为验收目标。

## 自动检查

- GSAU-Academic-Hub 的 16 项单元测试迁移后全部通过
- GSAU-Card 的 43 项单元测试迁移后全部通过
- 单元测试使用可提交且不含个人信息的本地样本
- Android Lint 无错误
- 教务系统登录与数据查询不使用明文网络
- Debug 与 Release 构建成功
- Release APK 使用正式签名并可安装
- 使用同一签名与更高版本号构建的 APK 可以覆盖安装上一版本

## 真机检查

- 启动后默认进入付款码页面，教务、付款码、设置三个底部入口及返回行为符合设计
- 甘农生活可与两个旧应用同时安装，且不会读取旧应用数据
- 校园卡链接绑定、多卡切换、余额、备注、重新绑定、删除和付款码刷新可用
- 付款码按 30 秒刷新、60 秒失效，全屏付款保持最高亮度与屏幕常亮
- 桌面付款组件支持切换校园卡、手动刷新和进入全屏付款页，添加后开启持续刷新并每 30 秒显示新付款码
- 教务系统与学工系统分别完成认证，任一会话失效不影响另一套系统
- 退出教务身份会同时清除两套学校会话与全部教务缓存
- 成绩、课表、考试与排名完整显示现有信息和交互
- 已查询的成绩分项明细可离线查看，并显示数据更新时间
- 断网或会话失效时显示带更新时间的已有教务数据，主动退出后清除教务缓存
- 统一认证取消、失败或进程重建后仍能返回原先选择的教务功能
- 当前学期能够从学校页面识别，开学日期和当前教学周可按学期设置并随日期递增
- 重新登录教务系统后保留开学日期，主动退出教务身份后清除
- 跟随系统、浅色、深色三种外观均正确显示统一灰绿色
- 按需刷新与持续刷新均可用，持续刷新达到系统时限后隐藏过期付款码
- 按需刷新不会启动前台服务或创建周期网络请求
- Android 13 及以上拒绝通知权限后，持续刷新仍能运行
- 设备重启后组件隐藏过期码，打开甘农生活后恢复持续刷新
- 设置页身份状态、系统设置入口、关于信息与手动更新检查可用
- API 24、API 34、API 35 与 API 37 设备覆盖核心流程
