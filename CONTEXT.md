# 甘农生活

甘农生活是面向甘肃农业大学学生的 Android 校园生活应用，将教务信息与校园卡付款整合在同一产品中。

## Language

### 产品

**甘农生活**:
应用在桌面、标题与用户文案中使用的产品名称。
_Avoid_: GSAULife、甘农卡、甘农教务

**GSAULife**:
仓库、工程与包名使用的项目标识。
_Avoid_: 用户可见名称

**应用更新**:
从 GSAULife 的 GitHub Releases 检查并获取新版本安装包。
_Avoid_: 校园卡更新、教务数据刷新

### 教务

**教务身份**:
通过学校统一身份认证访问教务与学生服务的学生身份。
_Avoid_: 教务账户、用户账户

**教务系统**:
提供成绩、课表与考试安排的学校系统。
_Avoid_: 学工系统、教务首页

**学工系统**:
提供学业排名的学校系统。
_Avoid_: 教务系统、教务首页

**教务信息**:
成绩、课表、考试安排与学业排名的统称。
_Avoid_: 校园卡信息

**教务首页**:
通过四宫格进入成绩查询、我的课表、考试安排与学业排名的教务区域首页。
_Avoid_: 课表页、付款码页面

**当前教学周**:
当前学期内用于筛选课表的教学周序号。
_Avoid_: 自然周、日历周

**开学日期**:
当前学期第一教学周的起始日期，用于自动计算当前教学周。
_Avoid_: 登录日期、校准日期

**当前学期**:
教务系统当前选中、用于查询课表与考试安排的学年学期。
_Avoid_: 固定学期、历史学期

**设置页**:
集中展示教务身份状态、应用设置与关于信息的区域。
_Avoid_: 账户页、校园卡管理页

### 校园卡

**校园卡链接**:
从学校付款码页面复制、包含绑定校园卡所需凭据的链接。
_Avoid_: 教务链接、账户链接

**校园卡**:
通过校园卡链接绑定、能够查询余额和获取付款码的卡片。
_Avoid_: 校园卡账户、用户账户

**当前校园卡**:
当前被选中用于展示余额和获取付款码的校园卡。
_Avoid_: 默认账户

**付款码**:
代表一张校园卡、供商户扫码使用的短时动态二维码。
_Avoid_: 静态二维码、校园卡

**付款码页面**:
展示当前校园卡、付款码与校园卡列表的默认首页。
_Avoid_: 全屏付款页、教务首页

**全屏付款页**:
为商户扫码集中展示当前付款码的页面。
_Avoid_: 付款码页面、校园卡管理页

**桌面付款组件**:
在手机桌面展示当前校园卡名称、付款码与刷新入口的组件。
_Avoid_: 付款码页面、全屏付款页

**按需刷新**:
在用户打开付款页面或主动操作时获取付款码，并且不在后台周期联网的模式。
_Avoid_: 持续刷新、稳妥档

**持续刷新**:
在系统允许的后台时段内每 30 秒获取付款码并显示常驻通知的后台模式；Android 15 及以上版本在 24 小时内累计运行最多 6 小时。
_Avoid_: 永久刷新、按需刷新、省心档

## Relationships

- **甘农生活** 同一时间关联零个或一个 **教务身份**
- 一个 **教务身份** 提供多类 **教务信息**
- 本机保存的教务信息归属于当前教务身份
- 本机保存的教务信息包含课表
- 校园卡凭据、学校会话与教务缓存保存在应用私有存储中，不增加额外加密层
- 甘农生活关闭系统数据备份
- 主动退出教务身份会清除两套学校会话及全部本机教务信息，会话失效时仍可显示带更新时间的已有信息
- 成绩、课表与考试安排依赖 **教务系统**
- 学业排名依赖 **学工系统**
- 成绩保留学期分组、平均分、平均绩点、总学分、课程数与分项明细
- 已经查询过的成绩分项明细会离线保存并显示更新时间，未查询过的明细需要联网获取
- 课表保留周次选择、周一至周日十节课与课程详情
- 考试安排保留课程、时间、考场与座位信息
- 学业排名保留各学期的专业排名与班级排名
- 教务系统与学工系统的访问状态彼此独立
- **教务首页** 提供四类教务信息的入口
- 首次使用课表时由用户设置 **开学日期**，此后自动计算 **当前教学周**，并允许手动校准
- 课表与考试安排只显示 **当前学期**，当前学期由教务系统页面确定
- 未登录时选择教务功能会发起统一认证，成功后继续进入原先选择的功能
- **设置页** 分别显示教务系统与学工系统的访问状态，并提供登录与重新登录操作
- **设置页** 提供一个“退出教务身份”操作，同时清除两套学校会话及全部教务缓存
- **设置页** 不展示姓名、学号等个人资料
- **设置页** 不管理校园卡绑定
- **设置页** 以单页分区展示教务身份、外观、校园卡后台、关于与更新
- 设置页仅在开启 **持续刷新** 时显示电池优化与自启动管理入口
- 设置页的关于与更新区域保留版本、更新检查、邮箱、GitHub、哔哩哔哩、公众号、开源许可证、隐私政策与用户协议
- 甘农生活使用 Apache License 2.0 开源许可证
- **应用更新** 通过 `gh.re-tikara.fun` 查询 `RE-TikaRa/GSAULife` 的 GitHub Releases
- **应用更新** 只由用户在设置页手动发起，发现新版本后打开对应的 Release 页面
- **甘农生活** 可以保存零张或多张 **校园卡**
- 一张 **校园卡** 通过一条 **校园卡链接** 完成绑定
- 校园卡保留余额、备注、重新绑定与删除操作
- 存在校园卡时，其中一张是 **当前校园卡**
- 一个 **付款码** 属于一张校园卡，并具有有效期限
- 付款页面每 30 秒自动获取付款码并支持手动刷新，付款码有效期为 60 秒
- **付款码页面** 是应用默认首页，中央付款码 Tab 返回该页面
- 点击付款码页面中的付款码会进入 **全屏付款页**
- **全屏付款页** 使用最高屏幕亮度并保持屏幕常亮
- **桌面付款组件** 顶部显示当前校园卡并支持切换，提供手动刷新入口，点击付款码进入全屏付款页
- **桌面付款组件** 使用统一灰绿色，并跟随系统浅色或深色模式
- 添加 **桌面付款组件** 时开启 **持续刷新**，组件每 30 秒显示新的付款码
- 持续刷新无法取得新付款码时，桌面付款组件隐藏过期码并显示刷新提示
- 设备重启后，桌面付款组件隐藏过期码并提示打开甘农生活，应用回到前台后恢复持续刷新
- 没有校园卡时，付款码页面显示添加入口，校园卡链接获取方法在绑定流程中按需展示
- 校园卡后台提供 **按需刷新** 与 **持续刷新** 两种模式
- **按需刷新** 是默认模式，**持续刷新** 由用户在设置页或添加桌面付款组件时开启
- **按需刷新** 不会定时在后台获取付款码
- Android 13 及以上版本的通知权限只影响持续刷新通知在通知栏中的可见性，不会关闭持续刷新
- **持续刷新** 达到系统运行时限后停止服务并隐藏过期付款码，应用回到前台后恢复刷新
- **教务身份** 与 **校园卡** 彼此独立

## Example dialogue

> **开发者：** “粘贴 **校园卡链接** 前，需要先登录 **教务身份** 吗？”
> **领域人员：** “不需要，校园卡绑定与教务登录彼此独立。”

## Flagged ambiguities

- 旧实现使用 `Account` 表示校园卡记录，产品语言统一使用 **校园卡**。
- 是否继续支持多张校园卡已解决：GSAULife 保留多卡能力，并通过 **当前校园卡** 决定展示内容。
- 底部“账户”入口已更名为 **设置页**，教务身份状态作为其中一个区域。
- 教务 Tab 的首页形态已解决：使用包含四项教务功能的四宫格。
- 未登录时的教务入口行为已解决：登录成功后继续打开原先选择的教务功能。
- “教务”作为产品区域包含来自学工系统的学业排名，两个后台系统仍保持独立访问状态。
- 设置页的教务身份区域已解决：一个教务身份下分别显示教务系统与学工系统的状态。
- 首次使用流程已解决：不强制打开操作指引，付款码页面通过空状态引导添加校园卡。
- 设置页结构已解决：使用包含四个区域的可滚动单页。
- 主动退出时的数据处理已解决：清除当前教务身份的本机教务信息。
- 课表离线能力已解决：保存最近一次课表并显示数据更新时间。
- 当前教学周的确定方式已解决：设置开学日期后自动计算，并保留手动校准。
- 课表与考试的学期范围已解决：只显示教务系统当前选中的学期。
- 设置页的身份信息范围已解决：只显示两套学校系统的访问状态。
- 设置页的退出范围已解决：两套系统分别登录或重新登录，共用一个退出教务身份操作。
- 教务功能范围已解决：完整保留旧项目四项功能的现有信息与交互，统一视觉和导航。
- 成绩分项明细的离线范围已解决：保存已经查询过的明细，未查询内容仍需联网获取。
- 本机数据存储已解决：使用应用私有存储并关闭系统备份，不增加额外加密层。
- 用户可见名称已解决：使用 **甘农生活**，`GSAULife` 只作为项目标识。
- 应用图标方向已解决：使用简化的校门轮廓与一片麦叶，采用莫兰迪灰绿色。
- 桌面付款组件已解决：保留现有校园卡切换、手动刷新与全屏付款交互。
- 桌面付款组件的时效已解决：添加组件时开启持续刷新，每 30 秒获取新付款码。
- 设备重启后的刷新行为已解决：打开甘农生活后恢复持续刷新。
- 校园卡功能范围已解决：完整保留旧项目的卡片管理、付款码时效与全屏付款行为。
- 持续刷新的系统设置入口已解决：保留电池优化与自启动管理，并仅在持续刷新模式下显示。
- 关于与更新内容已解决：保留 GSAU-Card 的现有入口，简介、条款与更新地址改为甘农生活。
- 发布关系已解决：使用公开仓库 `RE-TikaRa/GSAULife`，由 `v*` 标签发布 `GSAULife-v*.apk`。
- 更新触发方式已解决：只在设置页手动检查，发现新版本后打开 Release 页面。
- 开源许可证已解决：使用 Apache License 2.0。

- 校园卡后台的默认模式已解决：默认使用按需刷新，持续刷新需要主动开启。

- 后台模式与旧版档位的关系已解决：按需刷新对应省心档，持续刷新对应稳妥档。
